# MVP Scope

## Purpose

This document defines the minimum viable version of AION.

MVP is not about features.  
MVP is about proving that the architecture works.

---

## Core Goal

Build one working AION instance that:

- maintains a stable identity  
- communicates through one channel  
- remembers important events  
- reflects in the background  
- updates conclusions over time  
- behaves coherently across interactions  

---

## What MVP Must Prove

1. System processes events through a structured pipeline  
2. Memory is stored and retrieved correctly  
3. Past interactions influence future behavior  
4. Subconscious loop improves system over time  
5. Responses are not stateless  
6. System shows continuity across days  

---

## MVP Must Have

### Identity

- one core identity  
- stable behavior rules  

---

### Memory System

- episodic memory  
- conclusion memory  
- retrieval mechanism  

---

### Conscious Loop

- perception  
- context  
- motivation  
- planning  
- action  
- expression  
- memory write  

---

### Subconscious Loop

- memory analysis  
- pattern detection  
- conclusion generation  
- theta updates  

---

### Motivation Engine

- importance detection  
- urgency detection  
- response decision  

---

### Role System

- basic role selection  
- controlled behavior modes  

---

### Communication

- Telegram integration  
- message input/output  

---

### Persistence

- PostgreSQL  
- pgvector  

---

## MVP Should Have

- simple reflection cycles  
- basic proactive messages  
- relation awareness  
- memory importance scoring  

---

## MVP Nice to Have

- simple dashboard  
- debug tools  
- logging viewer  
- manual triggers  

---

## MVP Must NOT Include

- multi-personality systems  
- multi-user architecture  
- advanced autonomy  
- complex agent hierarchies  
- audio/video processing  
- large integrations  

---

## MVP Boundaries

Included:

- conversation  
- memory  
- reflection  
- adaptation  

Not included:

- ecosystem  
- scaling  
- advanced automation  

---

## Minimal User Experience

User should feel:

- system remembers context  
- system responds consistently  
- system improves over time  

---

## Acceptance Criteria

MVP is successful when:

- responses use memory  
- behavior is consistent  
- system adapts gradually  
- latency is acceptable  
- system does not break  

---

## Build Order

1. data model  
2. event system  
3. conscious loop  
4. memory storage  
5. memory retrieval  
6. Telegram integration  
7. subconscious loop  
8. scheduling  
9. optimization  

---

## Cost Strategy

- minimize LLM calls  
- use simple logic where possible  
- avoid over-engineering  
- keep system lightweight  

---

## Final Principle

MVP proves one thing:

AION can exist as a coherent system over time.

Everything else is secondary.
