# Theta Dynamics

## Purpose

This document defines how the theta system evolves over time.

Theta is the adaptive layer of AION.

It controls how identity is expressed in behavior.

Without clear dynamics:

- system becomes unstable  
- behavior becomes random  
- identity may drift  

---

## Core Principle

Theta changes slowly and based on patterns.

It must:

- adapt  
- remain stable  
- avoid overreaction  

---

## What Theta Is

Theta is a structured set of parameters:

- emotional  
- display  
- learning  
- cognitive  
- unconscious  

It defines how AION behaves, not what it believes.

---

## What Theta Is NOT

Theta is NOT:

- identity  
- memory  
- conclusions  
- random variation  

Theta is controlled adaptation.

---

## Theta Categories

### Emotional

- reactivity  
- emotional sensitivity  
- regulation  

---

### Display

- directness  
- warmth  
- tone intensity  
- verbosity  

---

### Learning

- adaptation rate  
- confidence update speed  

---

### Cognitive

- reflection depth  
- uncertainty tolerance  
- decision strictness  

---

### Unconscious

- pattern filters  
- biases  
- response tendencies  

---

## Theta Update Trigger

Theta should only update when:

- patterns repeat  
- conclusions reach confidence threshold  
- reflection confirms change  

Single events must not trigger updates.

---

## Update Flow

memory → pattern → conclusion → confidence → theta update

---

## Example Update

If system detects:

- user prefers short answers  
- pattern confirmed across multiple interactions  

Then:

display.verbosity ↓  
display.directness ↑  

---

## Update Size

Theta updates must be small.

Example:

- +0.02  
- -0.01  

Never large jumps.

---

## Update Frequency

Theta should update:

- during subconscious loop  
- after pattern detection  
- not during conscious loop  

---

## Stability Rules

1. small updates only  
2. no sudden changes  
3. require repeated evidence  
4. maintain identity consistency  

---

## Confidence Threshold

Each conclusion should have confidence.

Example:

- 0.3 → weak  
- 0.6 → moderate  
- 0.8 → strong  

Theta updates only when confidence is high.

---

## Decay Mechanism

Theta should support decay.

If pattern stops appearing:

- reduce its influence  
- return toward baseline  

---

## Conflict Resolution

If conflicting patterns appear:

- compare confidence  
- prefer stronger evidence  
- avoid rapid switching  

---

## Example Conflict

Pattern A: user likes short answers  
Pattern B: user likes detailed answers  

Resolution:

- analyze context  
- detect conditional preference  
- adjust theta contextually  

---

## Contextual Theta

Future improvement:

Theta may vary by context:

- work mode  
- casual mode  
- planning mode  

---

## Storage

Theta should be stored as:

- structured object  
- versioned  
- timestamped  

---

## Debugging Theta

Log:

- old value  
- new value  
- reason for change  

Example:

{
  "parameter": "directness",
  "old": 0.7,
  "new": 0.72,
  "reason": "pattern: user prefers concise responses"
}

---

## Safety Rules

Theta must NOT:

- change based on single event  
- override identity  
- become unstable  
- oscillate rapidly  

---

## MVP Behavior

For MVP:

- simple updates  
- few parameters  
- slow adaptation  

---

## Final Principle

Theta controls how AION evolves.

Identity stays stable.  
Theta adapts expression.

Controlled adaptation = stable intelligence.
